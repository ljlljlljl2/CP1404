# Reflection

## What I learned
In this practical, I learned how to create and use classes to model real-world objects such as cars, guitars, and programming languages. I also became more familiar with writing methods like `__init__`, `__str__`, and Boolean methods such as `is_dynamic()` and `is_vintage()`.

## What I found challenging
The most challenging part was understanding how to separate the class responsibilities from the client program. I needed to make sure that the class only handled its own data and behaviour, while input and output stayed in the main program.

## What I would improve next time
Next time, I would test each method earlier and more carefully while building the class. This would help me catch mistakes sooner and make the development process more efficient.

## Code review thoughts
Using branches and pull requests is becoming clearer with practice. Code reviews are useful because they help me see how other people read my code and what I can improve in naming, formatting, and structure.
