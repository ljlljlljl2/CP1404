# Practical 02 Reflection

## What did I learn from this practical?
In this practical, I learned how to use functions to make programs more organised and reusable. I also learned the importance of the Single Responsibility Principle, where each function should do one task only. In addition, I practised using Git and GitHub for version control, including committing, pushing and cloning repositories.

## What skills did I improve?
I improved my ability to break a program into smaller functions and write clearer code with docstrings. I also became more familiar with using PyCharm together with Git. My understanding of writing meaningful commit messages also improved.

## What was the most challenging part?
The most challenging part was understanding how to refactor existing code into functions without using global variables. It also took some care to make sure the score and temperature programs followed the required structure correctly.

## What will I do differently next time?
Next time, I will create functions earlier when writing a program instead of waiting until the end to refactor. I will also commit my work more regularly after each small milestone so that my version history is clearer and easier to manage.

## What am I still unsure about?
I am still a little unsure about when to separate code into more functions and how detailed commit history should be. I would like more practice with this so that it becomes more natural.
